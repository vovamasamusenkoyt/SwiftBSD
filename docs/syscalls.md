# Системные вызовы SwiftBSD

## Calling Convention

```
инструкция: syscall
номер:      RAX
arg1:       RDI
arg2:       RSI
arg3:       RDX
возврат:    RAX
затирается: RCX (RIP возврата), R11 (RFLAGS)
```

Ядро сохраняет RDI, RSI, RDX, RCX, R8, R9, R10, R11 на стек,
восстанавливает их перед `sysretq`.

errno **отсутствует** — ошибки возвращаются как отрицательные значения
в RAX (обычно -1). Userland проверяет `< 0`.

## Список syscall'ов

### 0 — `puts` (SC_PUTS)

```c
void puts(const char *s);
```

Печатает строку на последовательный порт (консоль).
Не возвращает ошибок.

### 1 — `nop` (SC_NOP)

```c
void nop(void);
```

Ничего не делает. Нужен для отладки.

### 2 — `yield` (SC_YIELD)

```c
void yield(void);
```

Отдаёт управление: `sti; hlt`. Позволяет таймерному прерыванию
переключиться на другой процесс.

### 3 — `open` (SC_OPEN)

```c
int open(const char *path, int flags);
```

Открывает файл на смонтированной файловой системе SwiftFS v2.

**Флаги:**
- `0` (`O_RDONLY`) — только чтение
- `1` (`O_WRONLY`) — только запись
- `2` (`O_RDWR`) — чтение и запись
- `0x100` (`O_CREAT`) — создать, если не существует
- `0x200` (`O_TRUNC`) — обрезать до нуля

**Возврат:** файловый дескриптор (≥ 0) или −1 при ошибке.

### 4 — `read` (SC_READ)

```c
int read(int fd, void *buf, uint32_t sz);
```

Читает данные из файлового дескриптора.

Для `fd == 0` (stdin): читает символы с последовательного порта
(блокирующий busy-wait, по одному байту за вызов).

**Возврат:** количество прочитанных байт (всегда `sz` для stdin).

### 5 — `write` (SC_WRITE)

```c
int write(int fd, const void *buf, uint32_t sz);
```

Пишет данные в файловый дескриптор.

Для `fd == 0` или `fd == 1` (stdout): выводит на последовательный порт.

**Возврат:** количество записанных байт.

### 6 — `close` (SC_CLOSE)

```c
int close(int fd);
```

Закрывает файловый дескриптор. Для fd 0 и 1 — no-op.

**Возврат:** 0 или −1.

### 7 — `exec` (SC_EXEC)

```c
int exec(const char *path, const char *args);
```

Загружает ELF64-исполняемый файл и передаёт управление.

**Процесс:**
1. Читает файл целиком в ядре (kmalloc)
2. Загружает ELF: PT_LOAD сегменты маппятся через vmm_map,
   выделяются 3 страницы стека (0x7F000000–0x7F002FFF),
   guard page (0x7F003000, не мапится), страница аргументов (0x7F004000)
3. Копирует строку аргументов в страницу 0x7F004000
4. Вызывает `user_entry(entry, 0x7F003000)` — `iretq` в ring 3

**Важно:** `exec` НЕ возвращается в caller при успехе — он замещает
текущий процесс целиком (перезаписывает адресное пространство).
При ошибке возвращает −1.

### 8 — `exit` (SC_EXIT)

```c
void _exit(int code);
```

Завершает текущий процесс.

**Процесс:**
- Сохраняет `exit_code` в структуре процесса
- Переводит процесс в состояние `PROC_ZOMBIE`
- Переходит в бесконечный цикл `sti; hlt` (ожидание `wait` от родителя)

### 9 — `fstat` (SC_FSTAT)

```c
typedef struct {
    unsigned short mode;
    unsigned long long size;
} stat_t;

int fstat(int fd, stat_t *st);
```

Возвращает информацию об открытом файле.

**Поля stat_t:**
- `mode` — тип/права (проверка `S_ISDIR(m)` через `m & 0040000`)
- `size` — размер в байтах

### 10 — `halt` (SC_HALT)

```c
void halt(void);
```

Останавливает систему: бесконечный цикл `hlt`.

### 11 — `meminfo` (SC_MEMINFO)

```c
int meminfo(uint64_t *buf);
```

Возвращает информацию о физической памяти.

**Формат:**
```
buf[0] = общий размер памяти в байтах
buf[1] = количество свободных страниц
```

### 12 — `fork` (SC_FORK)

```c
int fork(void);
```

Создаёт новый процесс (копирование текущего).

**Процесс:**
1. Клонирует PML4 через `vmm_clone_pml4` (copy-on-write для пользовательских страниц)
2. Выделяет новый стек ядра (4 KB)
3. Строит фрейм прерывания для возврата в user mode (child получит RAX=0)
4. Регистрирует процесс в планировщике

**Возврат родителю:** PID ребёнка (≥ 2)
**Возврат ребёнку:** 0
**Ошибка:** −1

### 13 — `wait` (SC_WAIT)

```c
int wait(int *status);
```

Ожидает завершения дочернего процесса.

**Процесс:**
- Если есть зомби-ребёнок: возвращает его PID, `*status = exit_code`
- Если есть живой ребёнок: вызывает `sched_yield()` в цикле, пока не появится зомби
- Если нет детей: возвращает −1

**Возврат:** PID ребёнка, 0 (ещё жив, не должно случаться), или −1.

### 14 — `getpid` (SC_GETPID)

```c
int getpid(void);
```

Возвращает PID текущего процесса (≥ 1).

## Память userland

```
0x08000000  ───── ELF code/data (PT_LOAD)
                  (размер из сегментов)
...               
0x7F000000  ───── стек: страница 0   (4 KB, RW, NX)
0x7F001000  ───── стек: страница 1   (4 KB, RW, NX)
0x7F002000  ───── стек: страница 2   (4 KB, RW, NX)
0x7F003000  ───── GUARD PAGE         (НЕ ЗАМАПЛЕНА)
0x7F004000  ───── args page          (4 KB, RW, NX)
                  RSP начальный = 0x7F003000
```

При загрузке ELF через `exec`: 3 страницы стека + guard + args page.
Суммарный стек = 12 KB. **Нет** `mmap`, `brk`, `sbrk` — без кучи. Всё,
что нужно программе, должно быть в ELF-сегментах. Аргументы командной
строки записываются в `args page` (0x7F004000).

## Структуры ядра

### `struct process`

```c
#define MAX_PROCS    64
#define KSTACK_SIZE  4096

struct process {
    int      used;
    int      pid;
    int      state;       // PROC_READY, PROC_ZOMBIE
    int      exit_code;
    int      parent;      // PID родителя
    uint64_t *frame;      // указатель на фрейм прерывания (контекст)
    uint64_t cr3;         // физический адрес PML4
    uint64_t kstack;      // физический адрес стека ядра (4 KB)
};
```

Планировщик: round-robin в `sched_irq_return`, вызываемом из
таймерного прерывания (`irq_handler` → `timer_irq`).
